## 为什么学 Docker？

联旌智能的 EaaS 平台底层就是容器化架构。他们自研的容器引擎虽然技术路线可能和 Docker 不同，但核心概念（镜像、容器、隔离、资源限制）是一致的。理解 Docker 的工作方式，是理解联旌产品的基础。

## 重点场景（结合联旌实际业务）

### 场景 1：大模型数据集挂载
高校要在容器里跑训练，数据集几十 GB，不可能拷进镜像。需要 `-v /data/dataset:/app/data` 把宿主机目录挂载到容器内。这和软链接 `ln -s` 解决的是同一类问题——**让程序访问到它"本该看到"的数据，而不是复制一份**。

### 场景 2：容器日志排查
容器跑崩了，怎么看原因？`docker logs <container>` 看标准输出，`docker inspect` 看容器元信息（端口、挂载点、环境变量）。这和用 `grep error` 看日志文件是一个逻辑，只是容器没有文件系统入口，必须通过 Docker CLI 来查。

### 场景 3：环境变量与 PATH 问题
容器里输入 `python`，调用的不是你刚 `pip install` 的那个——因为 PATH 顺序不对。这和在 Linux 上遇到的 Python 环境地狱是同一个问题。需要在 Dockerfile 里用 `ENV PATH` 显式指定。

### 场景 4：端口透传
容器内服务监听 8080，外部要访问到。`-p 8080:8080` 把容器端口映射到宿主机。如果访问不了，排查步骤：`ss -tlnp` 看宿主机端口起了没、`docker ps` 看端口映射对不对、`docker exec` 进容器看服务是否在运行。
