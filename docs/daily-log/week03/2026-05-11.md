## 核心认知

### Helm 是什么
- K8s 世界的包管理器（类似 apt/yum/pip）
- 用模板（`{{ .Values.xxx }}`）+ 变量（values.yaml）实现"一套配置多环境"
- 和联旌做客户交付的逻辑一致：一套 EaaS 平台，不同客户配不同 values

### 遇到的问题和解决
1. **Docker Hub 被墙** → 用阿里云镜像 + `--set image.repository` 覆盖
2. **sudo 导致 KUBECONFIG 丢失** → 复制 k3s.yaml 到 ~/.kube/config 永久解决
3. **ErrImagePull** → 用 `helm upgrade --reuse-values` 避免漏参数

### Helm 生命周期
```
install → deployed (REVISION 1)
upgrade → deployed (REVISION 2), REVISION 1 → superseded
rollback → deployed (REVISION 3), 内容是 REVISION 1 的配置
```
rollback 不是删除，而是创建新版本，和 git revert 逻辑一样。
