## 核心认知

### 1. Chunking 分块：固定大小 vs 按语义
- 固定 400 字符硬切会**切断知识点**（比如 Pod 解释被切成两半）
- 按 Markdown 标题切块保证**每个块是一个完整知识点**
- 企业里用 RecursiveCharacterTextSplitter：从大到小递归切（段落→行→词），和我们按标题切的思路一样

### 2. 检索距离值的含义
- ChromaDB 的 distance 值：越小越相似
- `< 0.5` 高度相关，`0.5-0.8` 有点相关，`> 1.0` 基本不相关
- 不加过滤的话，不相关的块也会交给 LLM，**干扰回答质量**
- 解决方案：多取几条 + 设阈值过滤

### 3. Prompt 工程：一字之差效果天差地别
- **"结合自身知识" = 给模型开了后门**，模型偷懒就用自己的知识回答，参考资料形同虚设
- 正确做法：强约束 Prompt —— "只使用参考资料"、"没有就说没有"、"不要编造扩展"
- 企业做法：System Prompt（定义行为） + User Prompt（实际内容）分离

### 4. Reranker 重排序
- **向量检索 = 图书管理员**（快速但粗糙），**Reranker = 教授**（慢但精准）
- 向量检索比较"向量空间距离"，细节被压缩；Reranker 同时读"问题+文档"做阅读理解
- 企业架构：向量检索从百万文档粗筛 20 条 → Reranker 精排 Top 5 → LLM 回答
- 模型推荐：BAAI/bge-reranker-v2-m3（中文效果好）

### 5. 完整 RAG 架构理解
```
文档加载 → 分块 → 向量化 → 存储
                                     ↓
用户提问 → 向量化 → 检索 → Reranker → 过滤 → Prompt → LLM → 回答
```
