## ✅ 完成情况

### 学了什么

1. **Agent 容器化实战**
   - 编写 Dockerfile：基础镜像 → 安装依赖 → 复制代码 → 启动命令
   - 理解 `--no-cache-dir` 减小镜像体积
   - 通过 `-e` 或 `--env-file` 在运行时注入 API Key，不打包进镜像
   - 踩坑：PyPI 下载慢，改用阿里云镜像源 `-i https://mirrors.aliyun.com/pypi/simple/`

2. **RAG 容器化诊断**
   - 问题：RAG 回答变笨，说"资料中没有相关信息"
   - 排查：data/ 目录放错了文件（放的是日志而不是知识库文档）
   - 修复：换成正确的 lianjing-business.md + tech-concepts.md，重新构建
   - 理解：Docker 容器里有什么取决于 COPY 了什么，容器内外文件系统隔离

3. **Docker 分层缓存原理**
   - 每行 Dockerfile 生成一个只读层
   - 构建时 Docker 对比内容变化决定是否复用缓存层
   - 先 COPY requirements.txt → RUN pip install → 再 COPY 代码，实现"代码改不重装依赖"

4. **5 周技能串联**
   ```
   Linux（第1周）→ 容器内运行的基础环境
   Docker（第2周）→ 构建镜像、容器化管理
   K8s（第3周） → 未来可编排部署多个容器
   RAG（第4周） → 知识库应用容器化
   Agent（第5周）→ 智能助手容器化
   ```

### 容易踩的坑

2. **容器化 RAG 数据丢失**：data/ 目录放错文件导致向量库为空
3. **交互式容器不加 -it**：input() 会直接报错退出

## 💡 今日感悟
容器化的核心价值不是"把程序跑起来"，而是**交付确定性**。不管换什么服务器、什么操作系统，只要装了 Docker，镜像跑起来的效果一模一样。这就是联旌 EaaS 平台的底层逻辑——把复杂的算力环境打包成用户开箱即用的服务。
