### Linux 复习

1. **文件权限模型**：`ls -la` 输出的第一列权限字符串（如 `drwxr-xr--`）的结构：
   - 第1字符：`d`=目录，`-`=普通文件，`l`=软链接
   - 后9字符分三组：所有者|组|其他人，每组 `rwx`
   - `chmod u+x script.sh` 为所有者添加执行权限
   - `./` 的必要性：当前目录 `.` 不在 PATH 中（安全考虑，防止路径劫持）

2. **grep / sed / awk 三剑客**（重点复习）：
   - **grep** = 找行：过滤包含指定模式的行。`-i` 忽略大小写，`-v` 反向匹配，`-c` 计数，`-r` 递归搜索
   - **sed** = 改行：流编辑器，按行做替换/删除/插入。`sed 's/旧/新/g'` 替换，`sed '/模式/d'` 删除，`-i` 原地修改文件
   - **awk** = 拆列：按分隔符拆成 `$1 $2 ... $NF`，可提取列、分组统计。`awk '{print $2}'` 打印第2列，`awk '/ERROR/ {print $1}'` 先过滤再提取
   - **管道组合**：`grep "ERROR" log | awk '{print $3}'` 典型的数据流处理模式

3. **管道（|）的本质**：前一个命令的标准输出（stdout）直接作为后一个命令的标准输入（stdin），不经过磁盘，内存中传递。这就是 Unix 哲学"每个程序只做一件事，用管道串联"。

### Docker 复习

1. **镜像分层原理**：Dockerfile 每条指令生成一层，层是只读的，多个容器可以共享同一个镜像的层。这就是为什么 `docker pull` 第二次很快——已存在的层不需要重新下载。

2. **Volume 的核心作用**：容器是临时的，容器一删，里面的数据就没了。Volume 把数据持久化到宿主机，容器内外通过挂载点共享数据。

3. **Dockerfile 最佳实践**：
   - `FROM` 尽量用 slim/alpine 镜像减小体积
   - `COPY requirements.txt` 和 `RUN pip install` 放在 `COPY . .` 之前，利用缓存层（代码变了但依赖没变时不需要重装）
   - `.dockerignore` 排除 `.git`、`__pycache__`、`.venv` 等不必要的文件

### 容易遗忘的点

- Linux 的 `.` 和 `..` 目录不是"特殊符号"，而是真实存在的目录项（`ls -la` 能看到）
- `sed -i` 是不可逆操作，一定要先用不加 `-i` 的版本预览
- awk 的 `END` 块只在所有行处理完后执行一次，适合做汇总统计
- Docker 的 `EXPOSE` 只是声明端口，不实际映射端口，要 `-p` 才能外部访问
