### Agent 复习核心知识点

1. **Agent 的本质**：大模型 + 工具 + 循环自主决策。和 Function Calling 的区别在于"谁来控制流程"——Function Calling 是代码（if-else），Agent 是模型（while 循环）。

2. **ReAct 范式**：Thought → Action → Observation → 循环 → Final Answer。每一步都存在 messages 中，模型知道"之前查过什么"。

3. **核心代码结构**：
   - `while turn < max_turns` 循环
   - 每次调用都要传完整的 `messages` + `tools`
   - 模型返回 tool_calls → 执行工具 → 追加 message + tool_result 到 messages
   - 模型不调工具 → 直接返回最终回答

4. **关键细节**：
   - `max_turns` 防止无限循环（模型可能卡住）
   - 每轮都要传 `tools`（API 无状态）
   - messages 必须追加完整的 message + tool_result（否则模型"失忆"）
   - API 要求 tool 角色消息必须有 tool_call_id
   - 工具结果过长需要截断

### toy_agent.py vs agent.py 对比

| 维度 | toy_agent.py | agent.py |
|------|-------------|----------|
| 工具来源 | 硬编码模拟数据 | DuckDuckGo 真实搜索 |
| 场景 | GPU 集群诊断 | 通用联网问答 |
| 核心循环 | 完全一样 | 完全一样 |

关键结论：Agent 框架和工具来源无关，核心 while 循环不用改。

### 容易遗忘点补充

- `tools` 参数不是"注册一次就行"，每次 API 调用都要传（HTTP 请求无状态）
- `messages.append(message)` 追加的是模型的 tool_calls 请求，`messages.append(tool_result)` 追加的是工具返回，这两个是一对
- 工具结果截断很重要（`agent.py` 用 2000 字符限制），否则上下文被塞满
- 工具描述的质量直接决定模型表现

## 感悟
复习 Agent 发现，最核心的是一个 while 循环。理解了这个循环，就能理解所有 Agent 框架（LangChain Agent、CrewAI 等）的本质——它们都是在这个 while 循环上做了封装和增强。面试时可以用 toy_agent.py 这个 60 行的最小实现来讲原理，比直接说"我用过 LangChain Agent"更有说服力。
