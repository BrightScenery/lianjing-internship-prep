## 核心认知

### 1. RAG 检索质量 = 数据质量 + 检索策略
之前 RAG 系统"很蠢"的根本原因是 data/ 目录里混入了大量周总结日志。这些日志以表格和格式标记为主，不是有效知识内容。向量检索时它们跟真正答案"抢排名"，导致检索失败。**企业级 RAG 系统必须做数据清洗和分类**——参考文档和学习记录不能混在一起。

清理后，5个测试查询（什么是RAG / Docker优势 / 联旌做什么的 / GPU MIG / RAG原理）全部命中正确来源。

### 2. .env 文件 = 密钥安全存储的标准方案
- `.env` 文件存密钥，被 `.gitignore` 保护
- `python-dotenv` 库自动将 `.env` 变量注入到 `os.environ`
- 代码里只写 `os.environ.get("KEY")`，不硬编码
- 换机器只需拷 `.env` 文件，不用改代码

### 3. Agent = 模型 + 工具 + 记忆 + 自主决策
普通模型被动问答，Agent 能自主规划行动（ReAct：Reason + Act）。Function Calling 是 Agent 的基础能力——让模型在需要外部数据时主动调用函数，而不是瞎编答案。
