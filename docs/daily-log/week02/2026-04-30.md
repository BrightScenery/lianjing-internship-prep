### Flask 应用编写

用 Python Flask 写了一个 Web 服务，包含 3 个端点：

| 端点 | 方法 | 返回 |
|------|------|------|
| `/` | GET | `{"message": "Hello from Flask in Docker!", "status": "ok"}` |

**踩过的坑：**
- `__name__` 写成 `_name_`（单下划线）→ Python 魔法变量必须双下划线
- `True` 拼写成 `Ture` → 运行时 NameError
- `methods=['Get']` → HTTP 方法必须大写 `GET`

### Dockerfile 理解

逐行理解了 Dockerfile 的含义：

```
FROM python:3.12-slim    ← 基于精简版 Python 镜像（150MB vs 完整版 1GB+）
WORKDIR /app             ← 创建工作目录（类似 mkdir + cd）
COPY requirements.txt .  ← 先拷依赖清单（利用缓存优化）
RUN pip install ...      ← 安装依赖（--no-cache-dir 减小体积）
COPY app.py .            ← 再拷应用代码
EXPOSE 5000              ← 端口声明（纯文档作用）
CMD ["python","app.py"]  ← 容器启动命令（exec 模式，不用 shell 模式）
```

**核心概念：Docker 分层构建**
- 每条指令是一层，Docker 会缓存每一层
- 文件没变时直接命中缓存，不用重新执行
- 所以 requirements.txt 和 app.py 要分开 COPY：改代码不重跑 pip install

### 容器生命周期管理

| 命令 | 作用 |
|------|------|
| `docker stop <name>` | 优雅停止容器（SIGTERM 信号） |
| `docker ps -a` | 查看所有容器（含已停止的） |
| `docker rm <name>` | 彻底删除容器实例 |
| `docker container prune` | 批量清理所有 Exited 容器 |
| `docker images` | 查看本地镜像 |
| `docker rmi <name>` | 删除镜像 |

**关键区分：**
- 容器 = 运行实例（临时的，删了不影响镜像）
- 镜像 = 安装包（持久的，删了不影响容器数据）
- 每次 `docker run` 创建一个新容器，即使用同一个镜像

## 💡 今日感悟

1. **Docker 的本质是隔离**：容器通过 Linux cgroup + namespace 实现进程级隔离，不需要虚拟机的完整 OS 开销
2. **Dockerfile 是标准化交付**：联旌工程师给客户部署 HPC 工具/AI 模型的流程，和我今天做的一模一样，只是换了 `FROM` 和 `RUN` 的内容
3. **国内网络是常态问题**：Docker Hub 被墙不是个别现象，联旌作为部署公司，每天也要处理这种网络问题，配置镜像源是基本功
4. **虚拟环境 vs 容器**：WSL 里装 venv 各种报错，但在 Docker 里 `pip install` 一次成功 — 容器就是隔离环境的终极形态
