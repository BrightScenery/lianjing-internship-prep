### 学了什么

1. **free**：查看内存。`-h` 人类可读，`-m` MB 单位。buff/cache 是页缓存，可被应用回收。
2. **df**：查看文件系统磁盘使用情况（分区级别）。`-h` 人类可读，`-hT` 带文件系统类型。
3. **du**：查看具体目录/文件大小。`-s` 只显示总计，`--max-depth` 控制深度。**与 df 的区别**：df 看分区整体，du 看具体目录占用。
4. **ps**：进程快照（静态）。`aux` 是 BSD 格式（带 %CPU/%MEM/STAT），`-ef` 是 System V 格式。
5. **top**：实时进程监控（动态刷新）。load average = 1/5/15 分钟系统平均负载。
6. **kill**：发送信号给进程。默认 SIGTERM(15)，`-9` 是 SIGKILL（强制杀），`-HUP` 重新加载配置。
7. **chmod**：修改文件权限。数字方式（755/644）和符号方式（u+x, g+w）。
8. **chown**：修改文件所有者。格式 `chown user:group file`，需要 root 权限，`-R` 递归。

### check-system.sh 的作用和意义

这个脚本模拟了运维中最常见的"服务器巡检"场景。联旌智能的 EaaS 平台需要管理多台 HPC 集群服务器，运维人员需要快速了解每台节点的健康状态。这个脚本把零散的系统命令整合成一份"巡检报告"，能在一分钟内输出：CPU 负载、内存、磁盘、进程分布、资源占用 TOP 进程、登录用户等关键指标。虽然目前版本还比较简单，但它是我理解"什么是自动化运维"的第一步——把重复性的手工检查变成一键脚本。

### 踩过的坑
- **问题**：`free -h | awk` 计算内存使用率得到 6842.1%
- **原因**：`free -h` 输出带单位（如 `7.6Gi`），awk 拿字符串做除法，把 `Gi` 忽略后当成数值运算出错
- **解决**：改用 `free -m`，输出纯数字（MB），awk 能正确计算
- **问题**：`du --max-depth=1 -h /` 一直没有输出
- **原因**：`/mnt/c` 挂载了 Windows 整个 C 盘（200G+），du 会扫进去，极慢
- **解决**：加 `--exclude=/mnt --exclude=/proc` 等，跳过挂载点
- **问题**：用 vim 编辑脚本后，输入 `:wq` 保存退出时没反应
- **原因**：中文输入法下输入的 `:` 是中文冒号，vim 识别不了
- **解决**：`:wq` 之前一定要先切换到英文输入法，再输入 `:wq`
