## 核心认知

### K8s 整体理解升级

之前学的是一个一个概念（Pod、Deployment、Service），今天把它们串起来了：

```
写 YAML → API Server 接收 → etcd 存储
                    ↓
      Controller 持续对比"期望 vs 实际"
                    ↓
      Scheduler 决定 Pod 去哪个节点
                    ↓
      kubelet 在节点上创建/管理 Pod
                    ↓
      kube-proxy 维护 Service → Pod 的网络映射
```

**一句话总结 K8s：** 你告诉它"我要什么状态"，它自动维持这个状态。

### 架构图里学到的东西

- **API Server 是唯一的入口**：kubectl、Helm、甚至 Web 界面，最终都调 API Server
- **etcd 是唯一的数据库**：所有配置和状态都存在这，丢了就全丢了
- **Controller 是"永动机"**：不停循环对比期望状态和实际状态，不一致就修
- **kubelet 是节点管家**：每个节点都有一个，向 API Server 汇报"我这儿正常"
- **kube-proxy 是网络管家**：Service 之所以能转发到 Pod，靠的是它写 iptables 规则
