### 学了什么
1. **大模型 API 调用的通用流程**：设 API Key → 构造 messages → 发请求 → 解析响应
2. **messages 格式**：所有大模型 API（DashScope、DeepSeek、OpenAI）都使用统一的 `{"role": "system/user/assistant", "content": "..."}` 格式
3. **System Prompt 的本质**：不是"第一句话"，而是给模型设定行为模式/运行参数。用户看不到，但会约束模型的回答风格
4. **多轮对话原理**：把完整对话历史（所有消息）都发给模型，模型基于上下文继续回答，而不是每次独立回答
5. **两个平台调用差异对比**：
   - DashScope：`Generation.call()` → `response.output.text`
   - DeepSeek：`client.chat.completions.create()` → `response.choices[0].message.content`
6. **Token 概念**：中文约 1 字 = 0.6~1 token。第一次调用消耗 630 token，多轮对话 977 token
