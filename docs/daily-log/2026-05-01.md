### 镜像分层原理理解

通过实际操作验证了 Docker 镜像分层的运行机制：

**UnionFS（联合文件系统）**：
- Docker 把多个只读层叠加成一个虚拟文件系统
- 每一层只记录和前一层的差异（增量）
- 容器启动后在最上面加一层可写层（容器层）
- 多个容器共享同一个镜像的底层，节省磁盘空间

**为什么分层重要**：
1. **缓存复用**：某一层没变时直接用缓存，不用重新构建
2. **镜像大小**：每层都会增加最终镜像体积，写得不好会从 100MB 膨胀到 500MB+
3. **安全**：每层永久保存在镜像中，在某层删除的文件仍可从历史恢复

### .dockerignore 文件

创建了 `.dockerignore` 文件，类似 `.gitignore`：
```
.venv
__pycache__
*.pyc
.git
.env
```
作用：防止不需要的文件被 COPY 进镜像，减小镜像体积。

### 层数压缩与缓存优化

理解了 Dockerfile 编写最佳实践：
- 把不常变的指令放前面（FROM, WORKDIR）
- 把常变的指令放后面（COPY 代码）
- 合并多个 RUN 指令减少层数（用 && 连接）
- 使用 `--no-cache-dir` 避免 pip 缓存进入镜像

##  今日感悟

1. **Dockerfile 不是写完就行，而是需要持续优化**：从"能跑"到"生产可用"之间，还有很多细节要注意，比如 .dockerignore、多阶段构建、非 root 用户等。
2. **镜像分层是 Docker 的核心优势**：理解了 UnionFS，就能明白为什么 Docker 比虚拟机轻量得多——不是虚拟化整个 OS，而是共享基础层。
3. **多阶段构建是生产环境标配**：联旌给客户交付镜像时，肯定不希望把编译器、开发工具也打包进去，只保留运行时必需的内容才是正确的做法。
