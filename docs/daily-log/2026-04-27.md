### 学了什么

1. **变量**：`name="value"`，`${var}` 界定变量名边界，`$(cmd)` 命令替换，`${var:-default}` 默认值
2. **条件判断**：`if/elif/else`，`[ -f ]` 文件判断，`[ $a -gt $b ]` 数字比较，字符串比较
3. **循环**：`for file in *.sh` 遍历文件，`while [ cond ]` 条件循环，`seq` 生成序列，`$((count + 1))` 算术运算
4. **函数**：`func() { ... }`，`$1 $2` 函数参数，`local` 局部变量，`return` 返回值，`$(func args)` 捕获函数输出

### batch-rename.sh 的意义

把今天学的变量、函数、循环、条件判断全部整合到一个实用脚本里。场景：给一批文件加前缀（运维中常见，比如给日志加日期前缀方便归档）。虽然功能简单，但它是我第一个真正意义上的"完整 Shell 程序"——有函数封装、有错误处理、有统计反馈，而不是零散命令的堆砌。

### 踩过的坑
- **`[ $name = "admin"]` 右括号前没空格**：`[` 是一个命令，需要空格分隔
- **`wc -l file` 输出包含文件名**：用 `wc -l < file` 或 `cat file | wc -l` 只输出数字
- **变量里的 `~` 不会自动展开**：`"~/hello.txt"` 字面路径不存在，要用 `$HOME/hello.txt`
- **`$fruits` vs `${fruit}s`**：shell 变量名边界问题
- **`((successed++))` 后++返回0=失败**：用 `((++successed))` 或 `successed=$((successed + 1))`
- **函数需要 return**：`if` 判断函数返回值时，函数内部要显式 `return 0/1`
