### 学了什么

1. **System Prompt 的本质不是"人设"，而是"运行参数"**
   - System Prompt 可以约束模型的行为模式、输出格式、知识边界、安全红线
   - 它不是"第一句话"，而是贯穿每一轮对话的最高级别指令
   - 模型是"概率生成"的，System Prompt 只能引导，不能强制执行

2. **4组 System Prompt 对比实验结果**
   - 无 system（基线）：模型自由发挥，回答全面但冗长，消耗 958 token
   - 老师人设：口语化、用比喻，用户体验好，消耗 725 token
   - 面试官人设：回答简洁（约150字 vs 基线 800+字），最后追问思考题，消耗 277 token
   - 极端格式约束（3句话，每句≤20字）：模型严格遵守，消耗仅 75 token
   - **结论：System Prompt 对输出长度的控制力直接关联成本，企业部署中 = 成本控制**

3. **User Prompt 写法对回答质量的影响**
   - 模糊提问（"怎么查看文件大小？"）：模型猜测意图，列出多种方案，消耗 812 token
   - 详细提问（含上下文和具体需求）：回答聚焦精准，消耗 547 token
   - **结论：问得越清楚，模型回答越聚焦，消耗 token 反而越少**

4. **System Prompt 与 User Prompt 冲突时的行为**
   - 自己实验中设置了矛盾的指令（system 说"不要解释"，user 说"说明区别和适用场景"）
   - 结果：模型倾向于满足 User Prompt 的具体要求
   - **结论：约束性指令放在 System Prompt 中，且不要与 User Prompt 矛盾**

### 思考题与回答

**Q1：为什么实验4的 Token 只有 75，而实验1有 958？**
A：实验1没有 system prompt，模型自由发挥尽量全面回答；实验4有严格格式约束（3句话、每句≤20字），模型必须简洁输出。Token 消耗与 system prompt 对输出长度的控制力直接相关。
补充：Token = 输入 + 输出，输入差异很小，输出差了 8 倍。System Prompt 对输出的控制力远大于它对输入的影响。

**Q2：老师人设相比默认回答的优缺点？从企业交付角度分析？**
A：优点：回答更符合预期、风格一致、可控。缺点：限制了创意性和多样性。企业场景下，联旌的客户（高校、科研院所）要的是稳定准确可控，不需要创意性，所以"缺点"在企业场景下反而是优点。
补充：System Prompt + RAG 组合才是企业级应用的标准配置。

**Q3：面试官 System Prompt 的约束有效吗？**
A：有效，回答比基线简洁了 80%。但模型没有完全遵守（仍加了 emoji 和格式化），因为 System Prompt 只能引导不能强制执行，模型是概率生成而非规则引擎。
