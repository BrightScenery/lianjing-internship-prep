### 环境变量与 PATH

**PATH 是什么**：环境变量，存储可执行文件搜索路径的列表。在终端输入 `python3`，shell 按 PATH 中的顺序依次查找，找到第一个匹配的就执行。

**关键操作**：
- `echo $PATH` — 查看当前 PATH
- `which python3` — 查看实际调用的是哪个路径下的 python3
- `export PATH="$HOME/mybin:$PATH"` — 把新路径加到 PATH 最前面，临时添加
- 永久生效：加到 `~/.bashrc`

**Python 环境地狱的根源**：系统 Python、Conda Python、Virtualenv Python 都在 PATH 里有路径。哪个先出现，输入 `python3` 就调用哪个。这就是为什么你明明装了某个包，`import` 却找不到——因为调用的是另一个 Python。


### 软链接与硬链接

**软链接（`ln -s`）**：类似 Windows 快捷方式，指向目标路径。目标删除，链接失效（死链）。可跨分区、可链接目录。

**硬链接（`ln`）**：和原文件共享同一个 inode（数据块）。目标删除，硬链接仍然可用。不能跨分区、不能链接目录。

**实际场景**：大模型数据集几 TB 在 `/data/dataset`，代码写死路径 `./data`。用 `ln -s /data/dataset ./data` 就能让代码访问到数据，不用拷。

## 💡 每日感悟

**在 Linux 中，文件后缀名（如 .txt）对系统没有任何强制语义。** 系统判断一个文件是什么，靠的是权限位和文件内容，不是后缀名。你完全可以把一个可执行脚本命名为 `hello.txt`，给它 `chmod +x` 后就能直接运行。后缀名只是给人看的约定，不是给系统看的标识。这一点和 Windows 完全不同——Windows 靠后缀名决定用什么程序打开，Linux 靠文件内容和权限。
