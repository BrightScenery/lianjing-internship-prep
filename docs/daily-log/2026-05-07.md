## 核心理解

### YAML 语法
- 缩进表示层级（2 空格，不能用 Tab），冒号后必须有空格，短横线 `-` 表示列表
- K8s 资源 YAML 固定骨架：`apiVersion` + `kind` + `metadata` + `spec`
- `metadata` 和 `spec` 是兄弟关系（同级第1层），不要把 spec 嵌套在 metadata 下

### 声明式 vs 命令式
- 命令式：`kubectl create deployment --image=nginx`，敲完就没了
- 声明式：`kubectl apply -f deployment.yaml`，配置可版本控制、可复用
- `apply` 是幂等的：资源不存在则 `created`，已存在则 `configured`

### imagePullPolicy
| 值 | 行为 | 场景 |
|---|---|---|
| `Always` | 每次远程拉取（`latest` 默认行为） | 生产环境 |
| `IfNotPresent` | 本地有就用，没有再去拉 | 本地开发 |
| `Never` | 只用本地镜像 | 离线环境 / 本地导入镜像 |

### k3s 镜像导入流程
Docker build → docker save tar → k3s ctr images import → 设置 imagePullPolicy: Never

## 踩坑记录
- `kubectl apply -f <文件>` 必须带 `-f` 参数，不加会报 "Unexpected args"
- `kubectl exec -it <pod> -- <命令>`，`--` 前后必须有空格，是 POSIX 分隔符
- 本地导入的镜像必须设 `imagePullPolicy: Never`，否则 K8s 默认去 docker.io 拉取，拉不到就 ImagePullBackOff
- Docker 和 k3s 镜像存储隔离：`docker images` 里的镜像 k3s 默认找不到
