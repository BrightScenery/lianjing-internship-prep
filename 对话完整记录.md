## 三、战略思考：为什么这样安排学习计划

> 以下内容不是"随便找了个教程开始学"，而是基于对联旌智能的业务模式、技术栈、以及 HR 直接反馈的深度研究后，**有意识地选择了这条路径**。

### 1. 研究了公司做什么
联旌智能的核心是：**把异构算力（CPU/GPU/NPU）通过容器化和调度技术，变成用户开箱即用的服务**。

### 2. 理解了需要什么样的人
```
底层能力：Linux 系统操作 + Shell 脚本
            ↓
中间层：Docker 容器化 + K8s 调度（公司自研容器引擎的基础）
            ↓
应用层：大模型部署 + RAG 知识库 + Agent（客户最常提的需求）
```

### 3. 为什么自下而上学习？
- **联旌不是应用层公司**，而是基础设施层公司。价值在于"让大模型跑起来"的底层能力。
- **HR 明确提到** K8s 和容器是核心，只是实习生初期碰不到核心代码。
- **定位**：大二实习生，短期不可能参与核心调度算法开发，但"理解容器调度原理 + 能独立完成大模型部署交付"就能在"打下手"阶段提供实际价值。

### 4. 计划与业务的映射
| 我学的 | 联旌的实际业务场景 |
|--------|-------------------|
| Linux + Shell | HPC 集群运维、客户现场实施部署 |
| Docker 容器化 | EaaS 平台底层就是容器化架构 |
| K8s 调度原理 | 公司自研多级调度技术的理论基础 |
| RAG 知识库 | 高校/企业最常见的 AI 落地场景 |
| 大模型部署 | 帮客户本地化部署 DeepSeek 等模型 |
| Agent | AI 应用发展方向，公司未来可能涉足 |
