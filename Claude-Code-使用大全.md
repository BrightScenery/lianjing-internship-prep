## 一、Claude Code 是什么

Claude Code 是 Anthropic 官方出品的**终端编程助手**。它不是在网页上对话，而是直接在你的命令行里运行，能：
- 读写你电脑上的文件
- 在终端里执行命令
- 搜索整个代码库
- 创建和管理 git 提交

**四个入口：**
| 入口 | 使用方式 |
|------|---------|
| 终端 CLI | 打开终端输入 `claude` |
| 桌面应用 | Mac/Windows 原生应用 |
| 网页版 | claude.ai/code |
| IDE 扩展 | VS Code / JetBrains 插件 |

---

## 二、核心能力

**我能做的：**
- 读取、编辑、创建、删除文件
- 运行 Bash 命令（构建、测试、部署等）
- 搜索文件和代码（Glob / Grep）
- 创建 git 提交和 PR
- 启动子 Agent 并行处理任务
- 创建定时任务（cron）
- 搜索网页获取最新信息

**我不能做的：**
- 直接操作你的 IDE 图形界面
- 访问需要你手动登录的网站（除非用 WebFetch）
- 绕过你的 git hooks 或安全限制（除非你明确要求）

## 三、交互方式

### 1. 直接对话
直接说需求就行，比如：
- "把 check-system.sh 的第 3 行改成..."
- "搜索所有包含 `docker-compose` 的文件"
- "帮我提交今天的代码"

### 2. 斜杠命令
| 命令 | 功能 |
|------|------|
| `/help` | 查看帮助 |
| `/clear` | 清空对话历史 |
| `/commit` | 创建 git 提交 |
| `/review-pr` | 审查 PR |
| `/loop` | 定期重复任务 |
| 其他 skill | 根据安装的 skill 而定 |

### 3. `!` 前缀
在提示里输入 `!` 开头，我会把它当作 Bash 命令执行。比如 `! git status`

### 4. @ 前缀
`@文件名` 可以直接引用文件内容到对话中。

## 四、高效使用技巧

### 技巧 1：说清楚你要什么，不要说怎么做
**好：** "把 check-system.sh 从 5 项检查升级到 9 项，加上磁盘 TOP、进程 TOP、登录用户"
**差：** "在第 3 行后面加一个 echo，然后在第 5 行..."

告诉我目标，让我自己决定实现方式。

### 技巧 2：批量独立请求一次性发
如果几个任务互相不依赖，一次说完，我会并行处理：
"同时帮我做三件事：1) 读 A 文件 2) 读 B 文件 3) 检查 git 状态"

### 技巧 3：让我先读再改
修改代码前，我会先读文件。如果你看到我直接改，可以说"先读一下再改"。

### 技巧 4：任务大的时候让我先规划
说"帮我做 X，先规划一下"，我会列出步骤再动手，避免走弯路。

### 技巧 5：给我一个"角色"或"上下文"
**好：** "你是运维工程师，帮我写一个服务器巡检脚本"
**好：** "我刚学 Linux 第二天，解释一下 ps aux 的输出"
这样我会调整回答的深度和角度。

### 技巧 6：用模板减少重复
每天要写的日志、要提交的代码，我可以记住格式，你说一句"今天这样"就够了。

### 技巧 7：让我写框架，你来填
学习场景下，我可以创建填空题、练习框架，你自己在终端里跑命令后填答案。这样你是在"做"，不是"看"。

## 五、常见坑 & 怎么避免

### 坑 1：我写了不存在的代码
**现象：** 我说"已完成"，但实际代码有 bug 或没写全。
**解法：** 重要改动后让我跑一下验证（`bash script.sh`），或者自己看 diff。

### 坑 2：我说"读过了"但其实文件改了
**现象：** 对话早期读的文件，后面可能被外部修改了。
**解法：** 如果怀疑文件被改过，让我"重新读一遍"。

### 坑 3：过度自信
**现象：** 我可能会给出看似正确但实际上不工作的代码。
**解法：** 让我实际运行验证，而不是只看代码。

### 坑 4：上下文太长后遗忘细节
**现象：** 对话很长时，早期的细节可能不再直接影响我的回答。
**解法：** 重要信息存到记忆系统（就是我们现在做的这个），每次自动加载。

## 六、记忆系统

**位置：** `~/.claude/projects/C--Users-H1831/memory/`

**原理：** 每次对话开始，自动加载 `MEMORY.md` 索引文件（很小），按需读取具体记忆文件。

**你现在的记忆：**
- `daily-workflow-push-after-commit.md` — 每次 commit 后必须 push
- `lianjing-project.md` — 联旌智能项目背景、6周计划、每日流程

**怎么用：**
- 让我"记住 XXX"，我会创建记忆文件
- 让我"看看记忆里有什么"，我会列出来
- 让我"忘掉 XXX"，我会删除对应文件
- 你自己也可以直接编辑那些 `.md` 文件

## 七、权限系统

我会调用工具（读文件、跑命令等），有些操作需要你批准。

**如果你想让我更自主：** 可以在设置里调整权限级别，这样常用操作不需要每次都问你。

**如果你想更安全：** 保持默认模式，每个操作你都能看到并拒绝。

---

## 八、最佳实践总结

1. **把我当同事，不当搜索引擎** — 告诉我目标、背景、约束，我会自己找到路径
2. **小步快跑** — 每次给一个明确任务，做完再给下一个，比一次给 10 个任务效果好
3. **让我验证自己** — 写完代码让我跑一下，比只看输出可靠
4. **养成记忆习惯** — 学到的约定、流程、偏好，让我存起来，下次不用重复
5. **不满意就说** — 我不喜欢猜你的心思，直接说"不对"、"换个方式"、"太啰嗦了"
