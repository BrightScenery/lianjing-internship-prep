# 技术概念速查

## 什么是 RAG（检索增强生成）？

RAG = Retrieval-Augmented Generation。核心思想是：**让大模型在回答之前先查阅相关资料**。

普通大模型的问题：它只依赖训练时记住的知识。如果你问它公司内部的私有信息，它肯定不知道。

RAG 的解决方案：
1. 把你的私有文档（公司手册、产品说明等）存进向量数据库
2. 用户提问时，先从数据库里找出相关片段
3. 把这些片段和问题一起交给大模型："参考资料如下，请回答这个问题"
4. 大模型基于资料生成回答

**类比**：开卷考试。普通大模型是闭卷考试，全靠记忆。RAG 是开卷考试，可以翻书再作答。

## 什么是向量数据库（Vector Database）？

向量数据库是专门存储和检索"向量"的数据库。

**向量是什么**：把一段文本通过 Embedding 模型转换成一串数字（比如 1024 个浮点数）。这串数字就是向量，它编码了文本的语义含义。

**关键性质**：语义相似的文本，它们的向量在数学空间中的距离就近。比如：
- "狗很可爱" → [0.12, -0.45, 0.78, ...]
- "宠物狗很萌" → [0.15, -0.42, 0.80, ...]  ← 距离很近
- "今天下雨了" → [0.90, 0.33, -0.12, ...]  ← 距离很远

向量数据库的核心操作：给定一个查询向量，快速找到数据库中距离最近的 K 个向量。这就是"相似度检索"。

常见的向量数据库：ChromaDB（轻量，适合本地）、Pinecone（云端）、Milvus（大规模）。

## 什么是 Embedding 模型？

Embedding 模型是专门用来做"文本→向量"转换的 AI 模型。它不像 ChatGPT 那样生成文字，只负责把语义编码成数字向量。

常见 Embedding 模型：
- **text-embedding-v3**（通义千问/阿里）：1024 维，中英文都支持
- **bge-large-zh**（智源）：中文优化效果好
- **text-embedding-ada-002**（OpenAI）：1536 维，英文为主

## 什么是 Docker？

Docker 是一种**容器化技术**。核心概念：

- **镜像（Image）**：打包好的"程序 + 所有依赖"的压缩包。就像游戏光盘，里面有运行所需的一切。
- **容器（Container）**：镜像运行起来的实例。就像光盘装好后运行的游戏程序。
- **关键优势**：Build Once, Run Anywhere。在我的电脑上构建的镜像，在任何有 Docker 的地方（本地、云服务器、客户的服务器）都能以完全相同的方式运行。

## 什么是 Kubernetes（K8s）？

Kubernetes 是**容器编排平台**。

类比理解：如果 Docker 是管理单个容器的工具，K8s 就是管理成百上千个容器的"调度中心"。

核心概念：
- **Pod**：K8s 的最小部署单元，通常包含一个容器
- **Deployment**：定义"我要跑几个 Pod，用哪个镜像，多少资源"
- **Service**：定义"外部怎么访问这些 Pod"
- **Node**：实际的服务器/虚拟机，Pod 跑在上面

为什么需要 K8s？因为生产环境中，你不可能手动管理几百个容器的启动、停止、故障恢复。K8s 自动处理这些。

## 什么是 GPU MIG 切分？

MIG（Multi-Instance GPU）是 NVIDIA A100/H100 等高端 GPU 的特性。

**问题**：一块 A100 GPU 很贵（几万元），但一个用户的任务可能只需要 1/7 的算力。如果整块卡只给一个人用，就浪费了。

**MIG 方案**：把一块物理 GPU 切成最多 7 个独立的小 GPU 实例，每个实例有自己的显存和计算核心。多个用户可以共享一块卡，互不干扰。

这对高校/科研院所特别有价值——实验室买了一块 A100，可以同时给多个研究生使用。
